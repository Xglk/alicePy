Introduction

This software aims to time activities by being simple, always available and remaining discreet about its presence on the screen.

The summary of the time spent on these activities allows us to better know the distribution of our time.

Installation

Retrieve the ChronoActivity zip.zip.

Unzip it somewhere on his PC.

In case it is an update, make sure to keep the Alice.db file of the old version to put it in the directory of the new version. In the absence of this file, at the first launch of the program, a new file Alice.db will be created blank of any data.

For an update, simply retrieve the last file located here: /lib/library.zip and run alice.exe again.

 

Throw

To launch the application, double click on the alice file.exe and that's it.

If you have Python and PyQt5 installed on your machine, the Alice.py script can be run instead, everything else will be of no use to you. You can rename it to Alice.pyw to avoid displaying a black command prompt window.

Add activities

A right mouse click on this icon displays the context menu

 

Start by selecting "Manage Activities", the following dialog box appears

 

In box 1 you enter the name of a new activity, validate it by the Enter key or by clicking on the Add button.

The activity will then appear in zone 2 which contains the list of all the activities listed.

With the left and right arrows (4) you can put the activities in zone 3 which represents all the selectable activities via the context menu that we will see below.

The order of these selectable activities can be changed with the up and down buttons (5).

An activity that is in Zone 2 may be deleted, provided that it is not present in Zone 3. To do this, select it and click on the delete button. This will not delete activities already counted

 

Existing activities can also be renamed. Everything is renowned, the history of actions, the menu, the corrections. If we rename to an already existing name, an alert is made, if we validate all the same, the two activities will then merge into one by cumulating their respective histories.

Start/stop an activity

Just click on the corresponding activity in the context menu to start it.

Starting a new one stops the previous one

Select the menu option "Stop activity" or "Exit" stops the current activity. If the current activity dates from a past day then it is necessary to go through the corrections to stop it. It is not planned to have an activity that runs on different days.

 

Visualize

Selecting the "Show calendar" menu brings up the following window:

 

It summarizes the activities and their time spent.

The calendar line displays the day in the year (from 1 to 365-366)

The correction line displays the days of the week, clicking on these days allows you to correct the times recorded on the corresponding day. Today's day is displayed in yellow. If a past day has an unfinished activity then the day is displayed in red.

The calendar can be displayed by the week or month

Totals can be set in hours/minute/second or daylights. During the day, it is considered that a full day is more than 5 hours, below it is half a day and below 1000 seconds, it is considered that the day has not yet begun.

"Always above" keeps the window above the others even when it is not active, it can be convenient when entering the CRA.

The box that can be entered above the "Next Period" button allows you to enter a date in different formats to access it quickly. An input aid is displayed if you hover the mouse over the input area.

 

Correct

Clicking on the day you want to correct in the calendar brings up this dialog box:

You can transfer a certain number of minutes from one activity to another

Stop the last activity at a given time if you forgot to stop the counter a day before

Add minutes to an activity. If necessary, this addition can be negative.

At the bottom it is a debug-like extraction of the data contained in the database for the day processed... Attention, the timestamps are displayed in GMT so it is necessary to take into account the time difference when defining the stop time or when analyzing the "debug" data
